# NARRATIVE: UNCERTAINTY ENGINE (Probability)

## 1. Core Concept
**Mathematical Concept:** 確率（場合の数、条件付き確率、期待値、漸化式）
**Omega Protocol:** "Uncertainty Quantification"
**Role:** オペレーターは **Probability Engineer（確率制御官）** として活動する。

不確実な事象の背後に潜む数学的構造を解析し、確率という数値で「未来の可能性」を定量化する。順列・組合せは「配置パターン」、条件付き確率は「情報フィルター」、期待値は「予測リターン」として解釈される。

## 2. Terminology Map
| 数学用語 | Omega Protocol 用語 | 説明 |
| :--- | :--- | :--- |
| **場合の数** | **Configuration Space** | 起こりうる全パターンの総数。 |
| **順列 (P)** | **Ordered Deployment** | 順序を区別する配置パターン。 |
| **組合せ (C)** | **Unordered Selection** | 順序を無視した選択パターン。 |
| **確率** | **Occurrence Likelihood** | 事象が発生する可能性の定量値。 |
| **条件付き確率** | **Filtered Probability** | 追加情報による確率の更新。 |
| **期待値** | **Projected Return** | 長期的に見た平均的な結果。 |
| **ベイズの定理** | **Inverse Inference** | 結果から原因を逆算する推論エンジン。 |

## 3. Mission Protocol: "The Forecast"

### Phase 1: 場合の数の計算（順列・組合せ）
*   **Briefing:** 「Configuration Space を列挙せよ。全パターンの把握なくして確率の制御はありえない。」
*   **Action:** 順列・組合せ・重複組合せの計算。最短経路、円順列。
*   **Success:** 「Configuration Space 展開完了。全 n 通りを確認。」

### Phase 2: 確率の計算と法則（加法定理・余事象）
*   **Briefing:** 「各事象の Occurrence Likelihood を算出し、確率法則を適用せよ。」
*   **Action:** 確率の加法定理、余事象、独立事象の計算。
*   **Success:** 「確率法則の適用完了。事象の発生確率を特定。」

### Phase 3: 高度な推論（条件付き確率・ベイズ・期待値）
*   **Briefing:** 「新たな情報を取得した。Filtered Probability を再計算し、Projected Return を評価せよ。」
*   **Action:** 条件付き確率、ベイズの定理、期待値、確率漸化式の計算。
*   **Success:** 「Inverse Inference 完了。UNCERTAINTY ENGINE の制御権を確保。」
